# dpik-workops frontend profile

- **Audience/job:** Internal operations staff monitoring and progressing DPIK work across repeated daily sessions.
- **Dominant archetype:** Dense operational console.
- **Signature behavior:** Exception-first queue explaining what changed, why attention is needed, and the safe next action.
- **Priorities:** Table/list scanability; saved filters; keyboard throughput; ownership/SLA; bulk safeguards; rollback; stale-data signals; export.
- **Verification:** Run build/lint; smoke dense desktop and tablet/mobile fallback, bulk recovery, permissions, stale/error states, and large-list responsiveness.

This profile is context, not a theme preset. Live repository instructions and product evidence override stale assumptions.
